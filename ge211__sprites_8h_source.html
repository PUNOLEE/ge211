56 // A `Texture_sprite` is a `Sprite` that can be rendered by copying
57 // a texture. Instead of specifying how to render themselves directly,
58 // derived classes must specify how to get a `Texture` representing
59 // the sprite. The dimensions of the sprite are the dimensions of the
60 // resulting texture. The return type of `Texture const&` means that
61 // `get_texture_` cannot just create and return a texture, but must
62 // store it somewhere. This is because `Texture`s will usually be
63 // cached. (Otherwise, you wouldn't use a `Texture_sprite`.)

74 };
